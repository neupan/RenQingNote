# 产品需求文档 (PRD)：人情记 (Flutter 纯本地版)

**文档版本：** v1.0 (纯本地单机版 MVP)
**应用形态：** iOS & Android 双端原生 APP
**核心技术栈：** Flutter + 本地数据库 (推荐使用 SQLite/sqflite 或 Isar)

## 1. 产品定位与核心策略

* **绝对隐私 (Privacy First)：** 所有账单数据仅保存在手机本地沙盒中，无需注册账号，无需网络即可使用，彻底打消用户对财务数据泄露的顾虑。
* **极速响应：** 由于没有网络请求的延迟，所有页面的加载、记账的保存、图表的渲染都必须做到“毫秒级”响应。
* **灾备机制 (核心痛点)：** 纯本地应用最大的风险是“换手机或卸载应用导致数据丢失”。因此，**本地数据的导出与恢复功能是 P0 级需求**。

## 2. 核心功能模块划分

APP 底部导航栏保留三个核心 Tab：**流水 (首页)**、**人脉**、**我的**。记账入口通过底部悬浮按钮 (FAB) 触发。

### Tab 1：流水 (首页)

**【功能定位】** 默认打开页，按时间轴展示所有人情收支记录。

* **顶部数据卡片：** 展示本年度“总收礼”与“总随礼”的本地汇总数据。
* **流水列表：**
* 按月份分组折叠展示记录。
* 记录字段：姓名、事件类型 (带图标，如结婚、满月)、日期、金额 (红色代表支出/随礼，绿色代表收入/收礼)。
* 交互：长按单条记录可弹出操作菜单【修改】或【删除】。



### 记账模块 (悬浮按钮触发)

**【功能定位】** 快速录入一笔往来账目。

* **双模切换：** 顶部切换【随礼 (支出)】与【收礼 (收入)】。
* **表单字段：**
* **金额 (必填)：** 调用纯数字键盘。
* **对象 (必填)：** 点击弹窗，从本地已有的“联系人”库中模糊搜索选择，或直接输入新名字创建新联系人。
* **事件 (必填)：** 提供常用 Tag (如结婚、压岁钱、丧葬、搬家等)，支持自定义。
* **日期 (必填)：** 默认今天，可呼出日历修改。
* **备注 (选填)：** 文本输入。



### Tab 2：人脉 (关系图谱)

**【功能定位】** 以“人”为维度，动态计算你与每个人的“人情盈亏”。

* **联系人列表：** 按姓名拼音 A-Z 排序。
* **智能对账显示：**
* 读取本地数据库，动态计算出当前联系人的差额。
* 例如：累计收张三 1000，随张三 500。列表中张三名字旁亮起绿色标签：`他欠我 ¥500`。如果账目持平则显示灰色的 `已平账`。


* **联系人详情页：** 点击进入后，仅展示与该联系人相关的所有历史收支明细。

### Tab 3：我的 (安全与数据管理)

**【功能定位】** 纯本地应用最重要的保障模块。

* **隐私锁 (App Lock)：** 支持开启每次打开 APP 强制进行面容 ID (FaceID) 或指纹解锁 (TouchID) 验证。
* **数据导出与备份 (重中之重)：**
* **生成 Excel/CSV：** 一键将所有数据生成 `.csv` 表格，调用系统分享面板，用户可发送到自己的微信或电脑上保存。
* **数据备份 (.json)：** 将整个 SQLite 数据库打包成一份 JSON 加密文件，保存在手机本地文件管理中。


* **数据恢复：** 支持从手机本地文件管理器中选择之前的 JSON 备份文件，覆盖恢复数据。

## 3. 本地数据库设计方案 (SQLite 核心表)

对于关系型的人情账本（人与账目强关联），强烈建议在 Flutter 中使用 `sqflite` 插件。我们需要建立两张核心表：

**表 1：`contacts` (联系人表)**

* `id` (INTEGER, 主键, 自增)
* `name` (TEXT, 姓名，重复则使用`memo`字段区分)
* `memo` (TEXT, 备注)
* `pinyin` (TEXT, 姓名的拼音，用于 A-Z 快速排序检索)
* `created_at` (INTEGER, 创建时间戳)

**表 2：`records` (流水记录表)**

* `id` (INTEGER, 主键, 自增)
* `contact_id` (INTEGER, 外键，关联 contacts.id)
* `type` (INTEGER, 0 代表随礼/支出，1 代表收礼/收入)
* `amount` (REAL, 金额，支持两位小数)
* `event_name` (TEXT, 事件名称，如“结婚”)
* `record_date` (INTEGER, 发生日期的精确时间戳)
* `note` (TEXT, 备注说明)

*计算盈亏逻辑：* 在人脉列表中，只需执行一条 SQL 聚合查询，按 `contact_id` 将 `type=1` 的金额之和减去 `type=0` 的金额之和，即可得出盈亏差额。

## 4. 开发落地步骤 (针对个人开发者)

1. **第一步 (环境搭建)：** 创建 Flutter 工程，引入核心依赖库：`sqflite` (数据库)、`path_provider` (本地路径)、`intl` (日期格式化)。
2. **第二步 (数据库封装)：** 编写一个单例模式的 `DatabaseHelper` 类，完成建表语句以及基础的增删改查 (CRUD) 方法。
3. **第三步 (核心 UI 开发)：** 完成“首页流水”和“记一笔”页面的 UI 拼装，并打通与本地 SQLite 的数据读取和写入。此时 MVP 的核心骨架已经跑通。
4. **第四步 (人脉与统计算法)：** 编写 SQL 聚合查询，完成“人脉”页面的盈亏计算和列表展示。
5. **第五步 (安全与备份)：** 引入 `local_auth` 实现生物识别锁；引入 `csv` 和 `share_plus` 插件实现数据的本地导出和分享。
